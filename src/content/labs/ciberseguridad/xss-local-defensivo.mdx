XSS no es "pegar un script en una web". Es un bug de contexto: un dato que debía ser tratado como texto termina siendo interpretado como HTML o JavaScript. En este lab lo ves en local, sin tocar sitios ajenos.

<Callout type="concepto">
  Tres palabras ordenan el problema: **source** es de dónde entra el dato,
  **sink** es dónde lo usás, y **contexto** define cómo lo interpreta el
  navegador.
</Callout>

## Página vulnerable de juguete

Creá un archivo local `xss-demo.html` con una idea como esta:

```html
<div id="saludo"></div>
<script>
  const params = new URLSearchParams(location.search);
  const name = params.get("name") ?? "mundo";
  document.querySelector("#saludo").innerHTML = `Hola ${name}`;
</script>
```

El source es `location.search`. El sink es `innerHTML`. El problema es que `innerHTML` interpreta HTML.

<TerminalBlock title="vt@labs:~/xss-local">

```bash
python3 -m http.server 8000
```

</TerminalBlock>

Abrí `http://localhost:8000/xss-demo.html?name=Valen` y después probá un valor con etiquetas HTML. No uses sitios de terceros: esto es tu archivo local.

## El fix mínimo

Si querés mostrar texto, usá una API que escriba texto:

```html
<script>
  const params = new URLSearchParams(location.search);
  const name = params.get("name") ?? "mundo";
  document.querySelector("#saludo").textContent = `Hola ${name}`;
</script>
```

Ahora el navegador no interpreta el contenido como HTML: lo muestra como texto.

<Callout type="error">
  Sanitizar a mano con reemplazos rápidos suele fallar. La primera defensa es
  elegir el sink correcto. Si no necesitás HTML, no uses `innerHTML`.
</Callout>

## Evidencia esperada

Documentá el bug con esta tabla:

```text
source:     query string (?name=...)
sink malo:  innerHTML
impacto:    input no confiable se interpreta como markup
fix:        textContent para renderizar texto
```

<Callout type="evidencia">
  Un buen writeup defensivo explica el flujo del dato y el cambio que elimina el
  bug. No necesita payloads contra servicios reales ni instrucciones ofensivas.
</Callout>
